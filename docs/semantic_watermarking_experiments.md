# Semantic Watermarking Experiment Workflow

This document describes the reproducible experiment layer in this unofficial fork. The upstream SFWMark code is preserved and credited to the original authors; the added scripts focus on fixed-key semantic watermarking, averaging/removal-attack evaluation, and METR-style experiments for a survey-paper research project.

This repository is not the official SFWMark repository. The official upstream project is [`thomas11809/SFWMark`](https://github.com/thomas11809/SFWMark).

## Repository Layout

```text
SFWMark/
  README.md
  requirements.txt
  install.sh
  scripts/
    run_paper_pipeline.sh
    run_fixed_key_pipeline.sh
  src/
    generate.py
    metric.py
    diff_attack/diff_wm_attack.py
    detect.py
    generate_fixed_key.py
    generate_fixed_key_fast.py
    detect_avg_attack.py
    utils.py
```

The generated image and latent folders are intentionally not tracked:

```text
outputs/
outputs_fixedkey*/
*.pt, *.npy, *.npz, *.tgz, checkpoints
```

Keep those files on the experiment machine or archive them separately.

## Environment

```bash
git clone https://github.com/bagpiper1458/Semantic-Watermarking.git
cd Semantic-Watermarking
conda create -n sfw python=3.10 -y
conda activate sfw
bash install.sh
```

The prompt datasets must be available under `src/text_dataset/` as described in the upstream README.

## Supported Methods

The experiment scripts support:

```text
Tree-Ring
RingID
HSTR
HSQR
METR
```

`METR` is an experiment method added in this fork. The other methods are inherited from the upstream SFWMark evaluation pipeline.

## Standard Multi-Key Pipeline

Run the full generation, quality metric, diffusion attack, and detection pipeline for one method/dataset:

```bash
bash scripts/run_paper_pipeline.sh HSQR coco outputs
```

Equivalent manual commands:

```bash
cd src

python generate.py \
  --wm_type HSQR \
  --dataset_id coco \
  --output_dir outputs

python metric.py \
  --wm_type HSQR \
  --dataset_id coco \
  --output_dir outputs

python diff_attack/diff_wm_attack.py \
  --wm_type HSQR \
  --dataset_id coco \
  --output_dir outputs

python detect.py \
  --wm_type HSQR \
  --dataset_id coco \
  --output_dir outputs \
  --no_save_inverted
```

Outputs are saved to:

```text
outputs/<dataset_id>/<wm_type>/
```

Important result files:

```text
clip.npz
fid.npz
verify-l1.npz
identify-acc.npz
metrics_<wm_type>.txt
```

## Fixed-Key Pipeline

The fixed-key setup evaluates large-scale deployment where all generated watermarked images use the same owner key. This is useful for averaging/removal-attack research.

Run a clean-only fixed-key experiment:

```bash
bash scripts/run_fixed_key_pipeline.sh Tree-Ring coco 0 outputs_fixedkey_k0
```

By default, the script reuses the clean image directory from:

```text
outputs/<dataset_id>/<wm_type>/img_pil
```

You can pass a different clean-image directory as the fifth argument:

```bash
bash scripts/run_fixed_key_pipeline.sh RingID coco 0 outputs_fixedkey_k0 /absolute/path/to/img_pil
```

Equivalent manual commands:

```bash
cd src

python generate_fixed_key_fast.py \
  --wm_type Tree-Ring \
  --dataset_id coco \
  --fixed_key 0 \
  --output_dir outputs_fixedkey_k0 \
  --shared_clean_dir ../outputs/coco/Tree-Ring/img_pil

python detect_avg_attack.py \
  --wm_type Tree-Ring \
  --dataset_id coco \
  --output_dir outputs_fixedkey_k0 \
  --only_clean \
  --max_trials 1000 \
  --no_save_inverted
```

Use `MAX_TRIALS` to reduce the detector runtime:

```bash
MAX_TRIALS=100 bash scripts/run_fixed_key_pipeline.sh Tree-Ring coco 0 outputs_fixedkey_k0
```

## External Attack or Averaging-Removal Evaluation

`detect_avg_attack.py` can evaluate images generated by an external attack/removal pipeline. Use folder overrides when the attacked clean/watermarked images do not live in the default output tree:

```bash
cd src

python detect_avg_attack.py \
  --wm_type Tree-Ring \
  --dataset_id coco \
  --output_dir outputs_fixedkey_k0 \
  --override_wm_dir /path/to/attacked_or_removed_wm_images \
  --only_clean \
  --run_tag avg_removal \
  --max_trials 1000 \
  --no_save_inverted
```

If both clean and watermarked folders are replaced, pass both:

```bash
python detect_avg_attack.py \
  --wm_type RingID \
  --dataset_id coco \
  --output_dir outputs_fixedkey_k0 \
  --override_clean_dir /path/to/clean_images \
  --override_wm_dir /path/to/attacked_wm_images \
  --only_clean \
  --run_tag external_attack
```

The detector writes:

```text
metrics_<wm_type>_<run_tag>.txt
verify-l1_<run_tag>.npz
identify-acc_<run_tag>.npz
```

Use `--skip_identification` when the attack output is only needed for verification or when 2048-way identification is too slow.

## Notes for GitHub Upload

Before pushing, check:

```bash
git status --short
git diff --stat
```

Do not stage generated folders such as `outputs/`, `outputs_fixedkey_k0/`, image folders, pattern lists, or tarballs. These artifacts are too large for Git and should be regenerated or stored outside the source repository.
